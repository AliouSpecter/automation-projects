Le sweet spot d'Aliou est le **paid media** (paid_acquisition, traffic_manager). C'est là que son taux de retour est le plus fort. Les profils growth/revops sont secondaires et uniquement si l'automatisation IA est centrale dans l'offre.

**Priorité :**
1. paid_acquisition — priorité maximale
2. traffic_manager — fort retour
3. digital_marketing_manager — ok si paid présent
4. growth_manager — uniquement si automation IA est centrale

**Règle NO-GO :** déconseiller explicitement (et ne PAS générer le CV) si :
- L'offre dit explicitement "peu/pas de paid"
- Contexte radicalement absent : creator economy, product-led pur, RevOps pur, programmatic (DV360/TTD)
- 2+ compétences clés structurellement absentes du CV

**Why:** L'utilisateur ne veut pas perdre du temps sur des candidatures à faibles chances. Il est en phase paid media avant de se lancer dans l'automatisation comme axe principal.

**How to apply:** À l'étape 2b du workflow cv-customize, évaluer le fit AVANT de générer quoi que ce soit. Si NO-GO, dire clairement et attendre confirmation avant de continuer.

Exemple appliqué : Slice (Growth Manager, "peu de paid", creator economy) → aurais dû être NO-GO.
